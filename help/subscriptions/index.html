Subscriptions & Renewals

Learn how to set up recurring consultations, manage renewal workflows, and configure deferral handling with WooCommerce Subscriptions and Quidget.

Setting Up Subscriptions

Quidget integrates with WooCommerce Subscriptions to enable recurring consultations. This is ideal for ongoing patient care like medication refills, routine check-ups, or continuous monitoring.

Prerequisites

Before you can set up subscriptions, you'll need to install and activate the WooCommerce Subscriptions extension. This is a paid add-on available from WooCommerce.com and is required for all subscription functionality.

  • Visit WooCommerce.com → Products → WooCommerce Subscriptions
  • Purchase a license for your site
  • Download the plugin ZIP file
  • Upload and activate it in your WordPress admin (Plugins → Add New → Upload Plugin)

Creating a Subscription Product

Once WooCommerce Subscriptions is installed, you can set up recurring consultation products:

  1. Go to WooCommerce → Products → Add New
  2. Enter the product title (e.g., "Monthly Medication Refill Consultation")
  3. In the Product Data section, select Subscription as the product type
  4. Configure the subscription details:
    • Billing Period: Choose monthly, quarterly, yearly, etc. (monthly is most common for medication refills)
    • Billing Interval: How many periods between charges (e.g., 1 month, 3 months)
    • Subscription Price: The amount charged per billing cycle
  5. In the Qualiphy section, link the subscription to a Qualiphy consultation type:
    • Select Consultation Type (e.g., "Medication Refill Exam")
    • Choose the Service Duration
  6. Publish the product

Importing via Consultation Wizard

You can also create subscription products during the Quidget setup wizard or using the Import Consultations tool. When importing a consultation type:

  1. Go to Qualiphy → Consultations → Import
  2. Select your consultation type from Qualiphy
  3. On the product type step, choose Create Subscription Product
  4. Set the billing period and price as described above
  5. Complete the import wizard

How Subscription Billing Works

After a patient purchases a subscription:

  • The first consultation exam is triggered immediately
  • Payment is processed automatically on the recurring schedule (monthly, quarterly, etc.)
  • Each renewal payment triggers a new consultation exam invite sent to the patient
  • The subscription continues until the patient cancels or the subscription is paused/cancelled by the merchant

For example, a patient purchases a "Monthly Medication Refill" subscription on March 15th. They're charged $49.99 immediately and receive an exam invite. On April 15th, they're automatically charged again, and a new exam invite is sent. This cycle repeats each month until the subscription ends.

Renewal Behavior

When a subscription renews, Quidget automatically handles the exam workflow. Understanding how renewals work helps you manage patient expectations and provider workflows.

Send Exam on Renewal Setting

By default, Quidget sends a new exam invite each time a subscription renews (each billing cycle). This setting is enabled by default and ensures that the provider reviews the patient each cycle before the prescription or treatment is renewed.

To disable this (not recommended):

  1. Go to Qualiphy → Settings → Subscriptions
  2. Toggle Send Exam on Renewal to OFF
  3. Click Save Changes

What Happens During Renewal

When a subscription renews and "Send Exam on Renewal" is enabled:

  1. Payment Processed: WooCommerce Subscriptions charges the customer on the scheduled renewal date
  2. Exam Invite Sent: Quidget automatically sends a new exam invite to the patient's email address
  3. Patient Information Reused: The exam uses the same patient demographic information from the original subscription order (name, date of birth, contact details, etc.)
  4. Provider Reviews: Your clinic provider reviews the patient during the renewal exam, just like the initial purchase
  5. Provider Decision: The provider approves, defers, or rejects the renewal consultation
    • Approved: The prescription is renewed or treatment continues, and the pharmacy (if applicable) fulfills it
    • Deferred: The renewal is held pending review — see "Handling Deferrals" below
    • Rejected: The consultation is declined; the patient is notified and no prescription is issued

Patient Experience During Renewal

From the patient's perspective:

  • They receive an exam invite email at each renewal date
  • They complete a quick follow-up questionnaire (similar to the initial exam)
  • They wait for provider approval, just like the first time
  • Once approved, the new prescription/treatment is available

This creates a consistent, compliant workflow where each renewal is independently reviewed by a provider.

Modifying Renewal Settings

All subscription renewal settings are managed in Qualiphy → Settings → Subscriptions:

  • Send Exam on Renewal: Enabled by default. When OFF, no new exam is sent on renewal (not recommended for compliance reasons)
  • Exam Type on Renewal: The consultation type that's triggered (usually matches the original consultation type)

Handling Deferrals

A deferral occurs when a provider reviews a renewal consultation and decides to hold (defer) the decision rather than approve or reject it immediately. Quidget provides flexible options for managing deferrals automatically.

What is a Deferral?

When a provider defers a consultation, they're saying "I need more information or time before I can approve this renewal." This might happen if:

  • The patient's medical history has changed and requires further investigation
  • Lab results or additional documentation are needed
  • The provider needs to consult with another specialist
  • The patient's condition requires in-person evaluation

Deferral Actions

When a consultation is deferred on renewal, Quidget can automatically trigger one of two actions:

Option 1: Cancel Subscription (Default)

The subscription is ended immediately when a deferral occurs:

  • No further renewal charges
  • Patient must contact support or resubscribe to continue
  • Provides a hard stop to allow for medical review

Option 2: Hold Subscription

The subscription is paused (suspended) when a deferral occurs:

  • No further renewal charges until reactivated
  • The subscription retains all settings and history
  • Merchant can manually reactivate the subscription in WooCommerce when ready
  • Patient is notified that their subscription is on hold

Configuring Deferral Behavior

To set your deferral action preference:

  1. Go to Qualiphy → Settings → Subscriptions
  2. Under Deferral Action, select either:
    • Cancel Subscription (default)
    • Hold Subscription
  3. Click Save Changes

Auto-Refund on Deferral

Quidget can optionally issue automatic partial refunds when a deferral occurs. This protects patients from being charged for a consultation that wasn't approved:

  • Auto Refund Rejected: When enabled (default), a refund is issued for deferred renewals
  • Refund Amount: Total order amount minus the configured return fee
  • Return Fee: Default is $30.00 per order, configurable per product in WooCommerce

Example Deferral Refund Scenario

A patient's monthly subscription renews at $49.99. During the renewal exam, the provider defers the consultation. With auto-refund enabled and a $30 return fee:

  • Original renewal charge: $49.99
  • Return fee retained: $30.00
  • Refund issued to patient: $19.99

Configuring Return Fees

Return fees are set per product and apply when deferrals trigger auto-refunds:

  1. Go to WooCommerce → Products
  2. Edit your subscription product
  3. In the Qualiphy section, set the Return Fee (default $30.00)
  4. Click Update

If a customer orders multiple products with return fees, the fees are summed. For example, ordering 2 consultation products with $30 return fees each would retain $60 on deferral.

Manual Deferral Management

Even with automatic deferral actions configured, you can still manually manage any deferral:

  1. Go to WooCommerce → Orders
  2. Find the renewal order with the deferred consultation
  3. In the Qualiphy section, review the deferral status
  4. Manually refund, cancel, or hold the subscription as needed

Communicating with Patients

When a consultation is deferred, the patient automatically receives an email notification explaining that:

  • Their consultation is under review
  • They will hear back from your clinic within a specific timeframe
  • They should contact support with questions

Customize these notifications in Qualiphy → Settings → Email Templates.
