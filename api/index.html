API Reference

The Quidget API allows you to manage consultations, handle webhook events, and integrate with the Qualiphy provider network.

Overview

Quidget communicates with the Qualiphy API to manage consultations. The plugin handles all API calls automatically — you don't need to write code. This reference is for advanced users who want to understand the data flow.

Base URL: api.qualiphy.me (used internally by the plugin)

Authentication

All requests to the Qualiphy API require authentication using your clinic's API key.

API Key

  • Found in your Qualiphy clinic dashboard → Settings → API
  • The key is stored in WordPress options (encrypted)
  • Each clinic location gets its own API key
  • The plugin validates the key on save and displays a green checkmark
Tip: Keep your API key secure and never share it publicly. Rotate your key periodically from the Qualiphy dashboard if needed.

Webhook Events

Quidget registers a webhook endpoint at your site. Qualiphy sends events to this endpoint when consultation status changes.

Supported Events

consultation.completed

Fired when a provider approves the consultation.

{
  "event": "consultation.completed",
  "consultation_id": "cons_abc123",
  "order_id": 1247,
  "patient": {
    "first_name": "Jane",
    "last_name": "D.",
    "state": "CA"
  },
  "provider": {
    "name": "Dr. Smith",
    "npi": "1234567890"
  },
  "exam_type": "gfe",
  "status": "approved",
  "completed_at": "2026-04-10T14:32:00Z"
}

consultation.deferred

Fired when a provider defers the consultation. An automatic partial refund is issued.

{
  "event": "consultation.deferred",
  "consultation_id": "cons_abc123",
  "order_id": 1247,
  "patient": {
    "first_name": "Jane",
    "last_name": "D.",
    "state": "CA"
  },
  "provider": {
    "name": "Dr. Smith",
    "npi": "1234567890"
  },
  "exam_type": "gfe",
  "status": "deferred",
  "reason": "Additional documentation needed",
  "deferred_at": "2026-04-10T15:15:00Z"
}

prescription.created

Fired when a prescription has been issued by the provider.

{
  "event": "prescription.created",
  "consultation_id": "cons_abc123",
  "prescription_id": "rx_def456",

Exam Invite Payload

When an exam invite is sent, the plugin POSTs the following data to Qualiphy's API:

{
  "api_key": "your_api_key",
  "exam_id": "exam_456",
  "patient": {
    "first_name": "Jane",
    "last_name": "Doe",
    "email": "jane@example.com",
    "phone": "+15551234567",
    "gender": "female",
    "dob": "1990-05-15",
    "state": "CA"
  },
  "order_id": 1247,
  "callback_url": "https://yoursite.com/wp-json/qualiphy/v1/webhook"
}

Field Descriptions

Field Type Description
api_key string Your Qualiphy clinic API key
exam_id string Unique exam identifier
patient.first_name string Patient first name
patient.last_name string Patient last name
patient.email string Patient email address
patient.phone string Patient phone (E.164 format)
patient.gender string Patient gender (male, female, other)
patient.dob date Patient date of birth (YYYY-MM-DD)
patient.state string Patient state (2-letter code)
order_id integer WooCommerce order ID
callback_url string Webhook endpoint for Qualiphy events

Consultation Statuses

Consultations progress through the following states:

Status Description
pending Exam invite sent, waiting for patient to connect
in_progress Patient connected with provider, consultation underway
approved Provider approved (GFE)
deferred Provider deferred the consultation
rx_pending Prescription being processed
rx_shipped Medication shipped to patient
rx_delivered Medication delivered
completed Final state (GFE approved or Rx delivered)
cancelled Consultation cancelled by patient or store

Error Codes

The API returns standard HTTP status codes. Common errors are listed below:

Code Meaning
401 Invalid API key
403 API key doesn't have permission for this exam type
404 Exam or consultation not found
422 Missing or invalid required patient fields
429 Rate limited (max 60 requests per minute)
500 Qualiphy server error (retry after 30 seconds)

Error Response Format

{
  "error": true,
  "code": 422,
  "message": "Missing required field: patient.email",
  "details": {
    "field": "patient.email",
    "issue": "Email address is required"
  }
}
