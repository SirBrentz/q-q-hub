Documentation

Settings reference and troubleshooting for the Quidget WordPress plugin.

General Settings

Found under Qualiphy → Settings → General. Controls the core plugin behavior.

Auto Send Exams
Default: Enabled
Automatically sends exam invites to the Qualiphy API when a patient completes checkout. Disable if you want to manually trigger invites from the order page.
Auto Launch Exams
Default: Enabled
Shows the exam launch modal (or redirect) on the WooCommerce thank-you page after checkout. When disabled, patients receive the exam link via email only.
Exam Launch Mode
Default: Popup
Choose between Popup (modal with meeting link, QR code, and dismiss option) or Redirect (countdown timer that sends the patient directly to the meeting URL).
Checkout Redirect Delay
Default: 3 seconds
Countdown duration before the exam launch modal opens or the redirect fires on the thank-you page.
Auto Refund Rejected
Default: Enabled
Automatically issues a partial refund when a consultation is deferred (not approved). The refund amount is the order total minus the return fee.
Exam Price Retained (Return Fee)
Default: $30.00
The default amount retained by the clinic when a consultation is deferred. Can be overridden per-product in the product editor. Per-item return fees are summed across all Qualiphy line items at checkout.
Default Price Margin
Default: 10%
Auto-fills the patient-facing price during consultation import based on the Qualiphy cost plus this margin percentage.

Checkout

The plugin automatically adds required fields to WooCommerce checkout for Qualiphy orders.

Additional Checkout Fields
Gender (select), Date of Birth (month/day/year selects), and phone number are added automatically when the cart contains a Qualiphy product. Phone is forced required.
Age Restriction
Rx consultations require the patient to be 18+. Non-Rx (GFE) consultations require 13+. Validated at checkout based on the DOB fields.
Account Creation
Guest checkout is disabled for all Qualiphy orders. The plugin forces WooCommerce account creation at checkout so patients can track their consultation status.
Auto Redirect to Checkout
Default: Enabled
Sends patients directly to checkout when they add a Qualiphy product to cart, skipping the cart page.

Consultations

Managed under Qualiphy → Consultations. This is where you import exams from Qualiphy and create WooCommerce products.

Consultation Types
GFE (rx_type 1): Good Faith Exam — provider clearance, no prescription. Order auto-completes on approval. Rx (rx_type 2): Full prescription consultation with pharmacy fulfillment and tracking.
Import Wizard
Creates or links WooCommerce products from Qualiphy consultations. Steps include: product type (simple/subscription), free/paid checkout, pharmacy selection (Rx only), variant configuration, pricing, and review.
Sync
Click the refresh button to re-sync consultations from the Qualiphy API. Syncs exam data, pharmacy information, and availability. 5-minute cooldown between manual syncs.

Pharmacy Routing

Found under Qualiphy → Pharmacies. Controls which pharmacy fulfills Rx prescriptions by state.

Routing Table
Maps exam IDs to pharmacy IDs by state code. When a patient checks out with an Rx consultation, the plugin looks up the correct pharmacy based on their billing state. If no routing rule exists, it falls back to any pharmacy that serves that state and exam.
Deep Sync
Fetches detailed pharmacy data from the Qualiphy API including state coverage and exam assignments. Runs automatically during setup and can be triggered manually.

Subscriptions

Requires the WooCommerce Subscriptions extension.

Send Exam on Renewal
Default: Enabled
Automatically sends a new exam invite when a subscription renews.
Subscription Deferral Action
Default: Enabled
When a consultation is deferred on a renewal order, automatically update the subscription status.
Subscription Deferral Status
Default: Cancelled
Choose whether to cancel or put on hold the subscription when a renewal consultation is deferred.

Email Notifications

Configure admin email notifications under Qualiphy → Settings → Emails.

Notification Recipients
Email addresses that receive admin notifications such as exam invite failures. Separate from patient-facing emails which are sent automatically by the Qualiphy platform.
Invite Failed Email
Sent to notification recipients when an exam invite fails during checkout. Includes the order number, error message, and retry instructions.

Branding

Configured under Qualiphy → Settings → Branding or during the setup wizard.

Clinic Logo
Separate uploads for light and dark backgrounds. PNG with transparent background recommended. Appears on the exam launch modal and patient emails.
Accent Colors
Default: Primary #CFC9FF, Secondary #6e54a7
Primary and secondary accent colors used for buttons, highlights, and the patient-facing consultation UI.

Service Area

Configured under Qualiphy → Settings → Service Area.

Supported States
48 U.S. states plus Washington D.C. are available. Alabama and Mississippi are currently excluded. The plugin creates a WooCommerce shipping zone ("Qualiphy Service Area") with free shipping for selected states. Patients in unselected states see a "not available in your area" message.

Language Settings

Customize all patient-facing text under Qualiphy → Settings → Language.

Customizable Text
Includes: Rx/non-Rx product messages, add-to-cart button text, already-in-cart messages, exam launch modal headings and descriptions, QR code text, email notes, free exam text, disabled exam messages, questionnaire questions, dose labels, and the consultation disclaimer.

Clinical Access

Controls who can view patient data in the WordPress admin.

Clinical Access Users
Only users with clinical access can view Qualiphy order details, patient exam IDs, and consultation results. Non-clinical users (marketing, development) see a "Patient Order Access Required" message instead. The admin who completes setup gets access automatically.

Troubleshooting

Common issues and fixes. Find your symptom below.

No payment gateway
Invites fire on payment confirmation, not order creation. Verify your gateway is active in WooCommerce → Settings → Payments and processing test orders successfully.
Caching interference
Exclude /checkout/ and /my-account/ from your cache plugin. If using W3 Total Cache, disable page cache for logged-in users.
Free exam misconfigured
For $0 consultations, ensure the product was imported with "Free Instant Checkout" in the import wizard. Free exams fire on order creation, not payment.
Account creation setting
Go to WooCommerce → Settings → Accounts & Privacy → enable "Allow customers to create an account during checkout."
Stale page cache
Clear your entire site cache after installing the plugin. Some caching plugins serve the thank-you page before the order session is established.
Auto-launch disabled
Qualiphy → Settings → General → enable "Auto Launch Exams." Also check launch mode (Popup vs Redirect).
Missing meeting URL
Check the WooCommerce order's Qualiphy section for invite status. If it says "error," the error message explains why (wrong state, pharmacy routing failed, etc).
Service area too narrow
Qualiphy → Settings → Service Area — check your selected states.
Shipping zone deleted
The plugin auto-creates a "Qualiphy Service Area" zone. If deleted, re-save your service area in Qualiphy settings to regenerate it.
Pharmacy routing gap
Qualiphy → Pharmacies — verify the exam has a pharmacy assigned for the patient's state.
Webhook blocked
Check if Wordfence, Sucuri, or a WAF is blocking incoming POST requests to the Qualiphy webhook endpoint.
