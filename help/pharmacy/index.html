Pharmacy & Fulfillment

Learn how Quidget routes prescriptions to pharmacies, configures state-level routing rules, and integrates shipment tracking.

How Pharmacy Routing Works

When a provider approves an Rx consultation, the prescription is automatically routed to a pharmacy partner. Quidget uses pharmacy routing rules to determine which pharmacy receives the prescription based on the patient's location and consultation type.

The Routing Map

The pharmacy routing table is found at Qualiphy → Pharmacies in your WordPress admin. It maps consultation types and states to specific pharmacy partners:

Consultation Type Patient State Pharmacy ID Pharmacy Name
GLP-1 (Ozempic) CA pharm_123 Golden State Pharmacy
GLP-1 (Ozempic) TX pharm_456 Lone Star Pharmacy
Testosterone CA pharm_789 Men's Health Pharmacy

How Routing Happens

  1. Patient purchases consultation: Checkout collects patient's state
  2. Provider approves exam: Quidget receives webhook with approval
  3. Prescription created: Quidget queries routing table for matching rule:
    • Look up consultation type + patient state
    • Find the corresponding pharmacy ID
  4. Pharmacy routed: Prescription sent to that pharmacy's fulfillment system
  5. Pharmacy processes: Pharmacy fulfills, ships, and provides tracking

Fallback Behavior

If no specific routing rule exists for a consultation + state combination, the system falls back to any pharmacy that serves that consultation type and state. This ensures prescriptions aren't stuck if a specific rule is missing.

Pharmacy Data Syncing

Pharmacies are synced from the Qualiphy API. When you run a consultation sync, pharmacy data is refreshed to include all available partners and their state coverage. New pharmacies added to your Qualiphy account appear in the routing table automatically.

Key Points

  • Each consultation type can route to different pharmacies by state
  • One patient state can have different pharmacies for different consultation types
  • Pharmacies are pre-integrated with Qualiphy—you don't add them manually
  • Routing happens automatically when a prescription is approved

Sync vs Deep Sync

Quidget provides two ways to refresh pharmacy data: regular Sync and Deep Sync. Each has different purposes and scope.

Regular Sync

A quick single API call to Qualiphy that fetches basic pharmacy information.

What It Does

  • Fetches list of available pharmacies
  • Updates pharmacy names and IDs
  • Updates general state availability
  • Updates pharmacy contact information

Speed

Fast—single API call, completes in seconds.

When to Use

  • After you first sync consultations during setup
  • When new pharmacies are added to your Qualiphy account
  • When pharmacy contact info or details change
  • Periodic refresh (monthly) to keep data current

How to Run

  1. Go to Qualiphy → Pharmacies
  2. Click Sync Pharmacies
  3. Wait for completion message

Deep Sync

A comprehensive multi-step process that fetches detailed pharmacy availability for each consultation and state combination. More thorough than regular sync, but slower.

What It Does

  • Fetches all available pharmacies
  • Queries detailed state availability for each consultation type
  • Verifies which pharmacies serve which states for which consultation types
  • Updates comprehensive routing rules
  • Builds complete coverage map for all consultation + state combinations

Speed

Slower—makes multiple API calls with 500ms delays between calls to avoid throttling. Has 5 calls per batch, so processing time scales with number of consultation types and states.

When to Use

  • After initial setup wizard to build complete routing rules
  • When you add new consultation types
  • When you expand to new states and want to verify pharmacy coverage
  • When troubleshooting pharmacy routing issues (see "wrong pharmacy" in troubleshooting)
  • Quarterly or semi-annually for comprehensive data refresh

How to Run

  1. Go to Qualiphy → Pharmacies
  2. Click Deep Sync Pharmacies
  3. A progress indicator will show during syncing
  4. Wait for completion (may take 1-3 minutes depending on consultation count)
  5. You'll see summary of updated routing rules

Regular Sync vs Deep Sync: Quick Reference

Aspect Regular Sync Deep Sync
Scope Basic pharmacy info Detailed routing rules
Speed Fast (seconds) Slower (minutes)
API Calls 1 call Multiple calls
Updates Routing General state level Per-consultation per-state
Use After Setup Monthly updates Quarterly, troubleshooting

Best Practice

Run a Deep Sync immediately after setup to build comprehensive routing rules, then use regular Sync monthly to keep pharmacy information current. Use Deep Sync again if you add new consultation types or need to troubleshoot routing issues.

Configuring State Routing Rules

You can manually override or configure pharmacy routing rules for specific consultation + state combinations. This is useful if you prefer a specific pharmacy for certain states or need to customize routing logic.

Accessing the Routing Table

  1. Go to Qualiphy → Pharmacies in your WordPress admin
  2. You'll see a table with Consultation Type, State, and Pharmacy columns
  3. This is your pharmacy routing configuration

Manual Configuration Steps

  1. Select a Consultation: From the routing table, find the consultation type you want to configure (or search for it)
  2. Choose a State: Select the state for which you want to set pharmacy routing (CA, TX, NY, etc.)
  3. Select a Pharmacy: From the dropdown, choose the pharmacy you want to route this consultation to in that state
  4. Save: Click Save. The rule is now active.

Editing Existing Rules

  1. Find the rule in the routing table (Consultation + State)
  2. Click Edit or the pencil icon
  3. Change the pharmacy from the dropdown
  4. Click Save

Example Scenarios

Scenario 1: Preferred Pharmacy in a State

You prefer a specific pharmacy in California for all GLP-1 consultations, even though multiple options are available. You set:

  • Consultation: GLP-1
  • State: CA
  • Pharmacy: Golden State Pharmacy (your preferred partner)

Now all approved GLP-1 prescriptions from CA patients route to Golden State Pharmacy.

Scenario 2: Consultation Type Variation by State

You offer HRT through two different pharmacies in different states:

  • Consultation: HRT | State: CA | Pharmacy: West Coast Wellness
  • Consultation: HRT | State: TX | Pharmacy: Lone Star Compounding

Depending on patient location, HRT prescriptions route to the appropriate regional pharmacy.

Viewing Current Routing

The routing table shows all active rules. You can:

  • Search: Filter by consultation type or state
  • Sort: Click column headers to sort by consultation or state
  • Bulk View: See all pharmacy options available for each consultation type

Adding New Rules

If a consultation + state combination doesn't have a routing rule yet, you can add one:

  1. Click Add New Routing Rule or the + button
  2. Select the consultation type
  3. Select the state
  4. Select the pharmacy
  5. Click Save

Default Behavior

If you delete a routing rule or leave a consultation + state combination without a rule, the system uses a default fallback: any pharmacy that serves that consultation type and state. This prevents prescriptions from being unroutable.

Shipment Tracking Integration

Once a pharmacy ships a medication, tracking information is automatically synced to the patient's WooCommerce order. Quidget supports multiple tracking integrations depending on your WordPress setup.

How Tracking Works

  1. Pharmacy Ships: Pharmacy partner fulfills and ships the prescription
  2. Webhook Received: Qualiphy sends a tracking_update webhook to your site with:
    • Tracking number
    • Carrier (USPS, FedEx, UPS, etc.)
    • Estimated delivery date
    • Order reference
  3. Quidget Processes: Quidget receives the webhook and stores tracking info
  4. Integration Method: Quidget adds the tracking number to the order using the best available method (see below)
  5. Order Updated: Order status set to Completed, tracking visible to customer
  6. Patient Notified: Email sent to patient with tracking link and carrier info

Integration Priority

Quidget uses the following priority to add tracking to orders (uses the first available method):

1. WooCommerce Shipment Tracking Plugin (Top Priority)

If you have the WooCommerce Shipment Tracking plugin installed, Quidget uses it. Tracking appears in:

  • Order detail page under "Shipment Tracking" section
  • Customer account → Order details → Tracking link
  • Automated tracking emails sent to customer

2. WooCommerce Order Fulfillment (WC 10.2+)

If you're running WooCommerce 10.2 or newer, Quidget uses the native Order Fulfillment feature. Tracking appears in:

  • Order detail page under "Fulfillment" tab
  • Customer account → Tracking info

3. Order Notes (Fallback)

If neither plugin nor native WC feature is available, Quidget adds tracking as a timestamped order note:

Shipment Tracking: [TRACKING_NUMBER] — [CARRIER] — [TIMESTAMP]

Setting Up Shipment Tracking

If Using WooCommerce Shipment Tracking Plugin

  1. Install and activate "WooCommerce Shipment Tracking" plugin from WordPress.org
  2. Go to WooCommerce → Shipment Tracking settings
  3. Configure carrier options and appearance
  4. Quidget will automatically use it for all tracking updates

If Using WooCommerce 10.2+ (No Plugin Needed)

  1. Update to WooCommerce 10.2 or newer
  2. No additional setup needed—Quidget will auto-detect and use the native feature

If Using Fallback (Order Notes)

  1. No setup needed—Quidget will automatically add tracking to order notes
  2. Tracking info is visible to staff in order detail page under order activity

Customer Experience

Depending on your integration method, customers see tracking in different places:

  • Shipment Tracking plugin: Dedicated tracking section with carrier logo and clickable tracking link
  • WC 10.2+ native: Fulfillment tab with professional tracking interface
  • Order notes fallback: Tracking appears in order timeline/activity section

Troubleshooting Tracking Issues

  • Tracking not appearing: Check that your site received the webhook (see Troubleshooting section). Verify plugin is installed if using Shipment Tracking.
  • Wrong carrier displayed: Ensure the carrier code from Qualiphy matches supported carriers in your tracking plugin
  • Tracking number incomplete: Verify the webhook payload includes full tracking number

Troubleshooting Pharmacy Issues

Common pharmacy and fulfillment problems and how to resolve them.

Issue: Prescription Not Created

Symptom: Provider approves exam, but prescription doesn't appear in order or pharmacy system.

Possible Causes

  • Patient is outside your service area
  • Pharmacy routing rule is missing for patient's state + consultation combination
  • Webhook delivery blocked or timing out
  • Exam approval webhook wasn't received by your site

How to Fix

  1. Check service area: Go to Qualiphy → Settings → Service Area. Verify patient's state is selected.
  2. Check pharmacy routing: Go to Qualiphy → Pharmacies. Search for the consultation type + patient state. If missing, add a routing rule or run Deep Sync.
  3. Check webhook delivery: Go to the order detail page. In the Qualiphy section, look for webhook status. If blocked, check security plugin settings (see Webhook section below).
  4. Manually resync: Go to Qualiphy → Consultations and click Refresh Consultations. This may trigger pending webhooks.

Issue: Wrong Pharmacy Selected

Symptom: Prescription routed to an unexpected pharmacy, not the one you configured.

Possible Causes

  • Pharmacy routing rule hasn't been set for this state
  • Multiple pharmacies serve the same state and fallback is being used
  • Routing rules are outdated and need deep sync refresh

How to Fix

  1. Check routing configuration: Go to Qualiphy → Pharmacies and search for the consultation + state.
  2. Set specific rule: If no rule exists, add one and select your preferred pharmacy.
  3. Run Deep Sync: Go to Qualiphy → Pharmacies and click Deep Sync Pharmacies. This refreshes all available pharmacy options and may reveal better routing options.
  4. Verify pharmacy availability: Confirm the pharmacy you want actually serves that state. Check with pharmacy or Qualiphy support.

Issue: Pharmacy Not Showing in Routing List

Symptom: Pharmacy you want to select doesn't appear in the dropdown when configuring routing rules.

Possible Causes

  • Pharmacy hasn't been synced yet
  • Pharmacy doesn't serve that consultation type or state
  • Pharmacy was removed from your Qualiphy account

How to Fix

  1. Run Deep Sync: Go to Qualiphy → Pharmacies and click Deep Sync Pharmacies. This will fetch detailed availability for all pharmacies.
  2. Check pharmacy coverage: Contact your pharmacy partner or Qualiphy support to confirm they serve your target consultation type + state.
  3. Verify pharmacy account: Log into your Qualiphy clinic dashboard and confirm the pharmacy is active and associated with your account.

Issue: Tracking Number Not Syncing

Symptom: Pharmacy ships medication but tracking number doesn't appear in WooCommerce order.

Possible Causes

  • Webhook payload doesn't include tracking number
  • Webhook delivery blocked by security plugin
  • Shipment Tracking plugin not installed or misconfigured
  • Order reference in webhook doesn't match WooCommerce order

How to Fix

  1. Check webhook delivery: Go to the order detail page. Look for webhook status in the Qualiphy section. If showing as blocked or failed, proceed to security plugin check.
  2. Check security plugin: If you use Wordfence, Sucuri, or similar, check that your site's webhook URL is whitelisted:
    • Go to Qualiphy → Settings → General
    • Copy the webhook URL shown (e.g., https://yoursite.com/wp-json/qualiphy/webhook)
    • In your security plugin, whitelist this URL
    • Test by asking pharmacy to resend tracking update
  3. Install Shipment Tracking plugin: If using fallback method, consider installing WooCommerce Shipment Tracking for better tracking UI.
  4. Check carrier format: Verify the carrier code from Qualiphy matches supported carriers (USPS, FedEx, UPS, DHL, etc.).

Issue: Webhook Blocked by Security Plugin

Symptom: Order shows "Webhook delivery blocked" or webhooks aren't being processed.

Possible Causes

  • Security plugin (Wordfence, Sucuri, etc.) is blocking requests from Qualiphy
  • Webhook URL not whitelisted in firewall settings
  • Rate limiting preventing webhook delivery

How to Fix

  1. Get your webhook URL: Go to Qualiphy → Settings → General. Copy the Webhook URL (usually /wp-json/qualiphy/webhook)
  2. Whitelist in Wordfence (if installed):
    1. Go to Wordfence → Firewall → Whitelisted IPs
    2. Add Qualiphy's webhook server IP (contact Qualiphy support for this)
    3. Go to Wordfence → Firewall → Whitelisted URLs
    4. Add your webhook URL
  3. Whitelist in Sucuri (if installed):
    1. Go to Sucuri Security → Firewall
    2. Add Qualiphy webhook server IPs to whitelist
  4. Check rate limiting: If your server has aggressive rate limiting, increase it to allow webhook delivery (discuss with hosting provider)
  5. Test webhook: After whitelisting, go to Qualiphy app and request a manual webhook test to verify delivery

General Pharmacy Troubleshooting Steps

  1. Check order detail page: Go to WooCommerce → Orders, open the order, and check the Qualiphy section for status and error messages
  2. Check webhook delivery: Look for webhook status and any error messages
  3. Run consultations sync: Go to Qualiphy → Consultations → Refresh Consultations to sync latest data
  4. Run pharmacy deep sync: Go to Qualiphy → Pharmacies → Deep Sync Pharmacies to refresh routing
  5. Check plugin logs: Go to Qualiphy → Logs (if available) to see API calls and errors
  6. Contact support: If issue persists, email support@qualiphy.me with the order number and any error messages

When to Contact Qualiphy Support

Reach out to Qualiphy support for:

  • Pharmacy not serving your state/consultation type
  • Recurring prescription creation failures
  • Webhook delivery issues that persist after security plugin checks
  • Pharmacy-side fulfillment problems (medication not shipped, wrong items shipped, etc.)
  • Questions about pharmacy partnerships or coverage

Email: support@qualiphy.me
