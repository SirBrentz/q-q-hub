Documentation

Settings reference and troubleshooting for the Quidget WordPress plugin.

General Settings

Found under Qualiphy → Settings → General.

Auto Send Exams
Default: Enabled
Automatically sends exam invites when a patient completes checkout. Disable to manually trigger from the order page.
Auto Launch Exams
Default: Enabled
Shows the exam launch modal on the thank-you page. When disabled, patients receive the link via email only.
Exam Launch Mode
Default: Popup
Popup (modal with meeting link + QR code) or Redirect (countdown that sends patient to meeting URL).
Auto Refund Rejected
Default: Enabled
Automatically refunds when a consultation is deferred. Refund = order total minus return fee.
Return Fee
Default: $30.00
Amount retained when deferred. Overridable per product. Summed across line items at checkout.
Default Price Margin
Default: 10%
Auto-fills patient-facing price during import based on Qualiphy cost + this margin.

Checkout

The plugin adds required fields automatically for Qualiphy orders.

Additional Fields
Gender, Date of Birth (month/day/year), phone. Phone forced required. Added when cart has a Qualiphy product.
Age Restriction
Rx: 18+. Non-Rx (GFE): 13+. Validated from DOB fields.
Account Creation
Guest checkout disabled. Plugin forces account creation so patients can track consultation status.
Auto Redirect to Checkout
Default: Enabled
Skips cart page when adding a Qualiphy product.

Consultations

Managed under Qualiphy → Consultations.

Types
GFE (rx_type 1): Provider clearance, no Rx. Auto-completes on approval. Rx (rx_type 2): Full prescription with pharmacy fulfillment.
Import Wizard
Creates or links WooCommerce products. Steps: product type, free/paid, pharmacy (Rx), variants, pricing, review.
Sync
Refresh button re-syncs from API. 5-minute cooldown between manual syncs.

Pharmacy Routing

Qualiphy → Pharmacies. Controls Rx fulfillment by state.

Routing Table
Maps exam IDs to pharmacy IDs by state. Falls back to any serving pharmacy if no rule exists.

Subscriptions

Requires WooCommerce Subscriptions.

Send Exam on Renewal
Default: Enabled
New exam invite on subscription renewal.
Deferral Action
Default: Cancel
Cancel or hold the subscription when a renewal consultation is deferred.

Email Notifications

Recipients
Admin emails for invite failures. Separate from patient emails (automatic).

Branding

Clinic Logo
Light/dark variants. PNG transparent recommended. Shows on exam modal and emails.
Accent Colors
Primary #CFC9FF, Secondary #6e54a7
Buttons, highlights, consultation UI.

Service Area

Supported States
48 states + D.C. AL and MS excluded. Auto-creates WooCommerce shipping zone. Unselected states show "not available."

Language

Customizable Text
Rx/non-Rx messages, button text, cart messages, exam launch modal, questionnaire questions, dose labels, disclaimer.

Clinical Access

Access Control
Only designated users see patient order data. Setup admin gets access automatically. Others see "Patient Order Access Required."

Troubleshooting

Common issues and fixes.

No payment gateway

Invites fire on payment confirmation. Verify gateway is active in WooCommerce → Settings → Payments.

Caching interference

Exclude /checkout/ and /my-account/ from cache. Disable object cache for logged-in users.

Account creation setting

WooCommerce → Settings → Accounts & Privacy → enable account creation during checkout.

Stale page cache

Clear entire site cache after plugin install.

Auto-launch disabled

Qualiphy → Settings → enable Auto Launch Exams.

Missing meeting URL

Check order Qualiphy section for error message.

Service area

Check selected states under Qualiphy → Settings → Service Area.

Shipping zone deleted

Re-save service area to regenerate the Qualiphy shipping zone.

Pharmacy routing gap

Verify exam has a pharmacy for the patient's state under Qualiphy → Pharmacies.

Webhook blocked

Check if security plugin blocks POST requests to the webhook endpoint.
