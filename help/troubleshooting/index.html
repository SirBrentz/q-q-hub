Troubleshooting

Step-by-step solutions for common Quidget issues. Most problems can be resolved quickly by following these guides.

6 articles in this section

Symptom: Patient completes checkout and payment processes successfully, but no exam invite email is sent.

Root Causes & Fixes

1. No Payment Gateway Active (Stripe 3D Secure / PayPal)

Exam invites fire on payment_complete, which is triggered after Stripe 3D Secure or PayPal authorization. If no payment gateway is configured, the trigger never fires.

Fix:

  1. Go to WooCommerce → Settings → Payments
  2. Verify at least one payment gateway is listed and marked as "Active"
  3. If none are active, enable one (Stripe is recommended):
    • Click on Stripe (or your preferred gateway)
    • Enable the payment method
    • Enter your API credentials
    • Click Save Changes
  4. Test a new order to verify invites are now being sent

2. Caching Interference

Caching plugins can prevent invites from firing. Exclude checkout and my-account pages from cache.

Fix:

  1. Go to your caching plugin settings
  2. Add these pages to the Cache Exclude List:
    • /checkout/
    • /my-account/
  3. Clear all caches
  4. Test a new order

3. Auto Send Disabled

Auto Send Consultations may be disabled in settings.

Fix:

  1. Go to Qualiphy → Settings → General
  2. Enable Auto Send Consultations
  3. Save changes and test a new order

4. Free Order Hook Conflict

Free orders (total $0) use woocommerce_order_status_completed hook instead of payment_complete. A plugin may be interfering.

Fix:

  1. Ensure Auto Send Consultations is enabled
  2. Disable other plugins temporarily and test
  3. The culprit is usually a payment gateway or subscription plugin

5. Invalid API Key

If your API key is invalid, invites cannot be sent.

Fix:

  1. Go to Qualiphy → Settings → General
  2. Your API Key should show a green checkmark
  3. If not, log into app.qualiphy.me
  4. Go to Settings → API and copy your API key
  5. Paste it into Quidget and verify
  6. Test a new order

Diagnostic Steps

Check the order's Qualiphy section for error messages:

  • Open the order in WooCommerce → Orders
  • Look for the Qualiphy section — any errors?
  • Check _qualiphy_invite_status meta on the order
  • Contact support with order ID and timestamp if still stuck

Symptom: After checkout, patient sees a login prompt instead of the order details page, exam invite, or thank-you message.

Root Causes & Fixes

1. Account Creation Not Enabled

WooCommerce has an option to allow guests to checkout without creating an account. Quidget requires customers to have accounts so they can access their orders.

Fix:

  1. Go to WooCommerce → Settings → Accounts & Privacy
  2. Find Allow customers to create an account during checkout
  3. Ensure this is CHECKED
  4. Also check that Allow customers to have a guest account is UNCHECKED
  5. Click Save Changes
  6. Clear all caches
  7. Test a new order as a guest

2. Stale Page Cache After Plugin Install

If you enabled Quidget after customers had already checked out as guests, cached pages may still show the old behavior.

Fix:

  1. Go to your caching plugin (WP Super Cache, W3 Total Cache, etc.)
  2. Click Purge All Caches or Empty All Cache
  3. Go to WooCommerce → Settings → Accounts & Privacy and re-save the settings (this clears WooCommerce caches)
  4. Clear your browser cache (hard refresh: Ctrl+Shift+Delete)
  5. Test a new order

3. Session Expired

If significant time passes between checkout and viewing the order (hours or days), the customer's session may have expired.

Fix:

This is normal behavior. Customers can log back in using the email and password they created at checkout:

  1. Direct them to My Account → Login
  2. They enter their email and password
  3. They can now view their order and access the exam invite

Prevent This Issue

  • Add a notice to your checkout page: "A login account is required to proceed. You'll use this to access your consultation exam."
  • Send a welcome email immediately after account creation with login details
  • Send a separate email with the exam invite link (this is automatic from Quidget)

Symptom: Patient reaches the thank-you page after checkout, but no exam popup or modal appears. They don't see a way to start the exam.

Root Causes & Fixes

1. Auto Launch Exams Disabled

By default, Quidget automatically opens an exam modal on the thank-you page. You may have disabled this.

Fix:

  1. Go to Qualiphy → Settings → General
  2. Find Auto Launch Exams
  3. Ensure it's toggled ON
  4. Click Save Changes
  5. Test a new order — the popup should now appear

2. No Meeting URL Generated

If the exam invite email didn't send properly, no meeting URL would be generated, so there's nothing to display.

Fix:

  1. Check the order in WooCommerce → Orders
  2. Look at the Qualiphy section
  3. If you see an error message, note it and contact support
  4. If there's no meeting URL listed, go back to the "Exam Invite Not Sending" section above

3. JavaScript Error or Conflict

A conflicting plugin or malformed theme code might be blocking the popup script.

Fix:

  1. Open the thank-you page in your browser
  2. Press F12 to open Developer Tools
  3. Go to the Console tab
  4. Check for red error messages — note them
  5. If there are errors, they often mention a specific plugin
  6. Disable that plugin and test again
  7. If you can't identify the culprit:
    • Go to Plugins
    • Deactivate all plugins except WooCommerce and Quidget
    • Test again — does the popup appear?
    • If yes, re-enable plugins one by one until the issue reappears

4. Exam Launch Mode Set to Redirect

You may have configured Quidget to redirect instead of showing a popup.

Fix:

  1. Go to Qualiphy → Settings → General
  2. Find Exam Launch Mode
  3. If it says "Redirect", change it to "Modal"
  4. Click Save Changes
  5. Test a new order — the popup should now appear instead of redirecting

Note: In Redirect mode, patients are sent to an external page instead of seeing a modal. This is intentional for some designs, but the default is Modal.

User Workaround

Even if the popup doesn't appear, patients can still access the exam by:

  • Checking their email for the exam invite
  • Going to My Account → Orders and clicking the order to see the exam link
  • Accessing the meeting URL directly if provided

Symptom: All customers see a "not available in your area" message on the store, regardless of their location. Consultation products can't be purchased.

Root Causes & Fixes

1. Service Area Not Configured

You may not have set up your service area, so Quidget has no states to allow.

Fix:

  1. Go to Qualiphy → Settings → Service Area
  2. Check the list of states — are any highlighted or selected?
  3. If not, click each state where you're licensed to practice
  4. Click Save Changes
  5. Refresh your storefront — customers should now see products in those states

2. Shipping Zone Deleted

Quidget uses WooCommerce shipping zones to determine service area. If the zone was accidentally deleted, availability breaks.

Fix:

  1. Go to WooCommerce → Settings → Shipping → Zones
  2. Look for a zone called "Qualiphy Service Area" or similar
  3. If it's missing, go back to Qualiphy → Settings → Service Area
  4. Re-save your service area selection (even if it hasn't changed)
  5. This will regenerate the shipping zone
  6. Check WooCommerce → Settings → Shipping → Zones again — the zone should now be present

3. WooCommerce Shipping Settings Misconfigured

Shipping rates or zone rules might be preventing the service area from working correctly.

Fix:

  1. Go to WooCommerce → Settings → Shipping → Zones
  2. Click on the Qualiphy Service Area zone
  3. Verify:
    • Zone Regions: Your states are listed
    • Shipping Methods: At least one method is enabled (Flat rate, Free shipping, etc.)
  4. If the zone looks empty or misconfigured, delete it and re-save your service area in Quidget (this recreates it)
  5. Refresh your storefront and test

Testing Service Area

To verify your service area is working:

  1. Go to your storefront as a guest (not logged in)
  2. Add a consultation to your cart
  3. Enter a shipping address in a state you serve
  4. The product should be available and purchasable
  5. Try a state where you don't serve — it should show "not available"

Permanently Excluded States

Alabama and Mississippi are currently permanently excluded from Qualiphy's network. Customers in these states will always see "not available in your area" regardless of your service area settings.

Symptom: Provider approves the consultation, but no prescription appears in the system or pharmacy gets no order.

Root Causes & Fixes

1. GFE Exam Type (Not Rx)

GFE (Good Faith Exam) exams (rx_type 1) don't create prescriptions — they're for clearance/approval only.

Fix:

  1. Check the consultation type — is it GFE or Rx?
  2. GFE types don't create prescriptions. This is expected behavior.
  3. If you need prescriptions, use an Rx consultation type instead

2. Pharmacy Routing Gap

Prescriptions can only be routed if a pharmacy is configured for the patient's state.

Fix:

  1. Log into app.qualiphy.me
  2. Go to Settings → Pharmacies
  3. Check the routing table for the patient's state
  4. If no pharmacy is assigned, add one for that state

3. Webhook Blocked by Security Plugin

Security plugins may block POST requests to /wp-json/qualiphy/v1/webhook.

Fix:

  1. Check for security plugins (Wordfence, Sucuri, etc.)
  2. Whitelist POST to /wp-json/qualiphy/v1/webhook
  3. Test prescription creation again

4. API Sync Delay

Sometimes there's a brief delay in syncing between Qualiphy and WooCommerce.

Fix:

  • Wait 5 minutes and refresh the order page
  • Check the order's Qualiphy section
  • If still no prescription after 5 minutes, contact support with order ID and timestamp

Webhooks are how Qualiphy notifies your site of important events like prescription approvals, exam completions, and renewals.

Webhook Requirements

For webhooks to work, your site must have:

  • Public accessibility: Not behind HTTP auth or IP restrictions
  • SSL certificate: HTTPS enabled (webhooks won't send to HTTP)
  • Correct webhook token/secret: Configured in your Qualiphy account

Security & Verification

Webhooks use HMAC-SHA256 signature verification. Each webhook request includes a signature that Quidget verifies to ensure it came from Qualiphy.

Rate Limiting

Webhooks are rate-limited to 60 requests per 60 seconds per IP. If you exceed this, requests will be queued or delayed.

Webhook Delivery & Cleanup

Auto-cleanup: Unprocessed webhooks are automatically deleted after 30 days. If a webhook fails repeatedly, it will eventually be discarded.

Common Webhook Problems

Webhook URL Cached

If your site uses page caching, the webhook endpoint may be cached, preventing successful delivery.

Fix: Exclude the webhook URL from caching in your caching plugin

Security Plugin Blocking POST

Security plugins may block POST requests to your webhook endpoint.

Fix: Whitelist the webhook URL in your security plugin

Site Behind HTTP Auth

If your site requires HTTP authentication (basic auth), webhooks cannot connect.

Fix: Disable HTTP auth or whitelist the Qualiphy IP range

Testing Webhooks

To test if webhooks are working:

  1. Complete a consultation and have the provider approve it
  2. Check the order page in WooCommerce within 5 minutes
  3. If prescription/status updates appear, webhooks are working
  4. If nothing updates after 5 minutes, webhooks may be blocked
