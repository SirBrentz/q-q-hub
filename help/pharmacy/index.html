Pharmacy & Fulfillment

Learn how prescriptions are routed to pharmacies, how to configure state-based rules, and how shipment tracking works in Quidget.

How Pharmacy Routing Works

Pharmacy routing is the system that automatically sends approved prescriptions to the correct pharmacy based on the patient's location. Understanding this system helps you ensure patients receive their medications efficiently.

The Routing Table

Access the pharmacy routing table from Qualiphy → Pharmacies in your WordPress admin. This table maps each exam type (consultation product) to pharmacy locations by state.

Example routing table:

Exam Type State Pharmacy
GLP-1 Starter California PharmaCare West
GLP-1 Starter Texas MediDirect Central
GLP-1 Starter Florida FastRx South
Testosterone California PharmaCare West
Testosterone All other states MediDirect Central (default)

The Routing Process

  1. Patient checks out: Enters their shipping address (state is recorded)
  2. Provider approves consultation: Creates a prescription in Qualiphy
  3. Prescription routed: System looks up the patient's state and exam type in the routing table
  4. Correct pharmacy identified: The prescription is sent to the appropriate pharmacy
  5. Pharmacy fulfills: Medication is prepared and shipped with tracking
  6. Tracking synced: Shipment tracking is sent back to your WooCommerce order

Default Fallback

If a state doesn't have a specific pharmacy assignment, the prescription is routed to your default pharmacy. You can set a default pharmacy in the routing table by assigning a pharmacy to "All other states" or "Default."

Example: If you have specific pharmacies for California and Texas, but no assignment for Nevada, Nevada prescriptions will be routed to your default pharmacy.

Multiple Exam Types

You can assign different pharmacies to different consultation products. For example:

  • GLP-1 consultations: Route to PharmaCare West (specialty GLP-1 fulfillment)
  • Testosterone consultations: Route to MediDirect Central (hormone specialty)

This allows you to use specialized pharmacies for different medication types while managing everything in one system.

Changing Pharmacy Assignments

If you need to change pharmacy routing (e.g., switch to a new pharmacy or add coverage for a new state):

  1. Go to Qualiphy → Pharmacies
  2. Edit the routing table by clicking on an assignment
  3. Select a new pharmacy from the dropdown
  4. Save your changes
  5. Future prescriptions will use the new routing immediately

Note: Changing routing doesn't affect prescriptions that have already been sent to a pharmacy. Only new prescriptions use the updated routing.

Configuring State Rules

State rules allow you to control exactly which pharmacy handles prescriptions in each state. This gives you fine-grained control over fulfillment and allows you to work with multiple specialized pharmacy partners.

Understanding State Rules

A state rule is a simple assignment: "For this exam type in this state, use this pharmacy." You can create as many rules as you need.

Example rules:

  • GLP-1 in California → PharmaCare West
  • GLP-1 in Texas → MediDirect Central
  • GLP-1 in all other states → MediDirect Central (default)

Adding a State Rule

  1. Go to Qualiphy → Pharmacies
  2. Click Add New Rule (or select an exam type to add rules for it)
  3. Choose the Exam Type (consultation product) from the dropdown
  4. Select the State from the list (or select "All other states" for a default)
  5. Choose the Pharmacy to route to
  6. Click Save Rule

Editing a State Rule

  1. Go to Qualiphy → Pharmacies
  2. Find the rule you want to edit in the routing table
  3. Click the rule to expand it
  4. Change the pharmacy assignment
  5. Click Save

Deleting a State Rule

  1. Go to Qualiphy → Pharmacies
  2. Find the rule to delete
  3. Click the Delete button (trash icon)
  4. Confirm the deletion

Note: If you delete a rule, prescriptions from that state will be routed to the default pharmacy (if one is configured). Make sure to set up a default before removing specific state rules.

Best Practices for State Rules

  • Always set a default: Configure a "default" rule (assign to "All other states") so prescriptions from unconfigured states still get routed
  • Review regularly: Periodically check your routing table to ensure it matches your current pharmacy partnerships
  • Test with one rule: Start simple with one pharmacy, then add state-specific rules as needed
  • Document your pharmacies: Keep notes on why each pharmacy is assigned to each region (e.g., faster shipping, specialty expertise)

Common Configurations

Simple Setup (One Pharmacy)

Easiest option for clinics with one pharmacy partner:

  • Create one default rule: GLP-1 → MyPharmacy (all states)
  • All prescriptions route to MyPharmacy regardless of state

Regional Setup (Multiple Pharmacies)

Good for clinics with regional pharmacy partnerships:

  • GLP-1 in West states (CA, OR, WA) → PharmaCare West
  • GLP-1 in Central states (TX, OK, NM) → MediDirect Central
  • GLP-1 in all other states → NationalRx (default)

Medication-Based Setup (Different Pharmacies per Drug)

For clinics with specialized pharmacy partners for different medications:

  • GLP-1 products → PharmaCare West (GLP-1 specialty)
  • Testosterone products → HormoneRx (hormone specialty)
  • Topical products → DermPharmacy (dermatology specialty)

Shipment Tracking

Once a prescription is routed to a pharmacy and fulfilled, Quidget automatically syncs shipment tracking information to your WooCommerce orders. Patients can track their medication deliveries directly from their account.

How Tracking Works

  1. Pharmacy ships: Pharmacy processes and ships the prescription with a carrier (USPS, FedEx, UPS, etc.)
  2. Tracking number generated: Pharmacy gets a tracking number from the carrier
  3. Webhook received: Quidget receives the tracking number via a Qualiphy webhook
  4. Order updated: Tracking number is added to the WooCommerce order
  5. Patient notified: Patient receives an email with the tracking number and carrier link

Viewing Tracking in WooCommerce

To see shipment tracking for an Rx order:

  1. Go to WooCommerce → Orders
  2. Click the order to open the detail page
  3. Scroll to the Qualiphy section
  4. Look for Shipment Tracking
  5. You'll see:
    • Carrier name (USPS, FedEx, UPS, etc.)
    • Tracking number
    • Link to track shipment

Viewing Tracking as a Patient

Patients can track their shipments from their account:

  1. Patient logs in to their account on your site
  2. Goes to My Account → Orders
  3. Clicks the order to view details
  4. Finds the Shipment Tracking section
  5. Clicks the tracking number to view real-time tracking on the carrier's website

WooCommerce Shipment Tracking Integration

Quidget integrates with the popular WooCommerce Shipment Tracking plugin (if installed). When tracking information syncs, it's automatically added to the order in Shipment Tracking format, making it visible in standard WooCommerce reports and order lists.

When Tracking Updates Are Delayed

Tracking information typically appears within minutes of shipment. If tracking is delayed:

  • Wait 24 hours: Pharmacy systems and carriers may have a 24-hour sync delay
  • Contact the pharmacy: The pharmacy may not have uploaded tracking to their system yet
  • Check webhooks: If tracking still isn't appearing, there may be a webhook issue (see troubleshooting below)

Tracking Information in Emails

When tracking updates, patients automatically receive an email containing:

  • Confirmation that their medication has shipped
  • Carrier name
  • Tracking number
  • Direct link to track the shipment
  • Estimated delivery date (from carrier)

You can customize this email message in Qualiphy → Settings → Language under "Shipment Notification Email."

Multiple Shipments

If an order contains multiple items, the pharmacy may ship them separately. In this case:

  • Each shipment gets its own tracking number
  • The order shows multiple tracking entries
  • Patients receive separate tracking emails for each shipment

Troubleshooting Pharmacy Issues

If you encounter problems with pharmacy routing, prescription creation, or shipment tracking, use this guide to diagnose and resolve the issue.

Issue: Prescription Not Created After Approval

Symptom: Provider approves an Rx consultation, but no prescription appears in Qualiphy or the pharmacy system.

Diagnosis

  • Check the patient's state in the order
  • Go to Qualiphy → Pharmacies
  • Look for a routing rule for that exam type and state

Solution

If no route found: Add a routing rule for that exam type and state.

  1. Go to Qualiphy → Pharmacies
  2. Click Add New Rule
  3. Select the exam type and state
  4. Choose a pharmacy
  5. Save and retry the approval

If route exists: The issue may be with your API key or pharmacy configuration. Contact support with the order ID and exam type.

Issue: Wrong Pharmacy Selected

Symptom: Prescription was routed to the wrong pharmacy for a patient's state.

Diagnosis

  • Check the order for the patient's state
  • Go to Qualiphy → Pharmacies and find the routing rule for that state
  • Verify the rule points to the correct pharmacy

Solution

  1. Edit the incorrect routing rule
  2. Change the pharmacy to the correct one
  3. For future prescriptions, the new routing applies automatically
  4. For already-shipped prescriptions, contact the pharmacy to see if rerouting is possible

Issue: Webhook Blocked / Tracking Not Updating

Symptom: Prescription is approved and sent to the pharmacy, but tracking information doesn't appear in WooCommerce orders for days.

Diagnosis

Webhooks are HTTP POST requests that Qualiphy sends to your site when events occur (approval, shipment, etc.). If your site has security plugins blocking these requests, updates won't be received.

Check Your Security Plugins

Common plugins that may block webhooks:

  • Wordfence Security
  • Sucuri Security
  • iThemes Security
  • Firewall plugins
  • Rate limiting plugins

Solution

  1. Log in to your WordPress admin
  2. Check any security plugins you have installed
  3. Look for logs or blocked requests sections
  4. Search for POST requests to wp-json/quidget/v1/webhooks/prescription-update
  5. If found, whitelist this webhook endpoint in your security plugin settings

Webhook endpoint to whitelist:

POST /wp-json/quidget/v1/webhooks/prescription-update

If you're not sure how to whitelist webhooks in your security plugin, contact support with your plugin name and we'll provide specific instructions.

Issue: Pharmacy Not Showing in Routing Table

Symptom: When trying to create a new routing rule, your pharmacy doesn't appear in the pharmacy dropdown.

Solution

  1. Go to Qualiphy → Settings → General
  2. Verify your Qualiphy API key is still valid (should show a green checkmark)
  3. If the checkmark is red, reconnect your API key
  4. Go back to Qualiphy → Pharmacies and refresh the page
  5. The pharmacy list should now populate

If the pharmacy still doesn't appear, contact support — it may need to be added to your Qualiphy clinic account first.

Issue: Tracking Number Not Received from Pharmacy

Symptom: Order shows "Shipped" status, but no tracking number appears in WooCommerce.

Diagnosis

The pharmacy may not have uploaded tracking to their system yet, or there may be a webhook issue.

Solution

  1. Wait 24 hours (pharmacies and carriers have delays)
  2. Check WooCommerce → Orders to see if tracking appeared
  3. If still missing after 24 hours, contact your pharmacy directly for the tracking number
  4. You can manually add tracking using the WooCommerce Shipment Tracking plugin

Issue: Patient Received Wrong Medication

What to do:

  1. Contact the pharmacy immediately with the order and tracking details
  2. The pharmacy can initiate a return and send the correct medication
  3. Document the issue in the order notes for your records
  4. Consider issuing a courtesy credit or refund if appropriate

Medication errors should be reported to your pharmacy partner immediately. Quidget routing is only responsible for sending the prescription to the correct pharmacy — the pharmacy is responsible for dispensing the correct medication.

Getting More Help

  • Email: support@qualiphy.me
  • Include: Order ID, exam type, patient state, and a description of the issue
  • For webhook issues: Include your security plugin names and any error messages from your logs
