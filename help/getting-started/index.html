Getting Started

Everything you need to know to install, activate, and configure the Quidget plugin on your WordPress site.

System Requirements Installing API Key Setup Wizard First Consultation

System Requirements

Before installing Quidget, ensure your WordPress site meets these minimum requirements:

Core Requirements

  • WordPress 5.0+ (WordPress 6.0+ recommended for best compatibility)
  • WooCommerce 5.0+ (WooCommerce 6.0+ recommended)
  • PHP 7.4+ (PHP 8.0 or higher recommended for optimal performance)

Additional Requirements

  • SSL Certificate: Your site must be served over HTTPS (SSL/TLS). This is required by the Qualiphy API and essential for patient security.
  • Public Accessibility: Your site must be publicly accessible on the internet. Webhooks from Qualiphy need to reach your server to sync exam status, prescription updates, and pharmacy fulfillment data.
  • Qualiphy Account: You need an active Qualiphy clinic account with API access. Sign up at app.qualiphy.me if you haven't already.

Payment Processing

  • WooPayments: Recommended for processing consultation fees. WooPayments integrates directly with WooCommerce and handles patient payments for consultations.
  • Other Payment Gateways: Quidget supports any WooCommerce payment gateway (Stripe, Square, etc.). You only need one to process consultation purchases.
  • Free Consultations: You can offer free consultations without a payment gateway—patients will be able to complete checkout and access exams without payment.

Optional but Recommended

  • WooCommerce Subscriptions: Required only if you offer recurring or subscription-based consultations. Not needed for one-time consultations.
  • WooCommerce Shipment Tracking: Optional plugin that displays pharmacy shipment tracking info in patient order emails and accounts.

Server Configuration

Your server should allow:

  • Outgoing HTTPS requests to api.qualiphy.me (for API calls)
  • Inbound POST requests from api.qualiphy.me (for webhooks)
  • PHP functions: curl_exec(), json_encode(), json_decode()

If you use a security plugin (Wordfence, Sucuri), you may need to whitelist Qualiphy's webhook URL. See Troubleshooting for more details.

Installing the Plugin

Quidget installs like any standard WordPress plugin. You can install it via the WordPress admin dashboard.

Installation Steps

  1. Log in to your WordPress admin dashboard
  2. Navigate to Plugins → Add New
  3. Click Upload Plugin at the top of the page
  4. Select the Quidget plugin ZIP file from your computer
  5. Click Install Now and wait for installation to complete
  6. Click Activate Plugin to enable Quidget on your site

What Happens After Activation

Once activated, Quidget will:

  • Add a Qualiphy menu to your WordPress admin sidebar
  • Auto-detect your WooCommerce installation
  • Display a setup wizard notice at the top of your admin dashboard
  • Create database tables for storing consultation and pharmacy data

Verify Installation

To verify successful installation:

  1. Look for the Qualiphy menu in your admin sidebar
  2. Click on Qualiphy → Settings → General
  3. You should see plugin settings fields. If the page loads without errors, installation is complete.

You're ready to move to the next step: connecting your Qualiphy API key.

Connecting Your Qualiphy API Key

The Quidget plugin communicates with your Qualiphy clinic account through an API key. This secure connection ensures that patient exams are routed correctly, consultation data syncs, and webhooks are processed properly.

Step 1: Find Your API Key in Qualiphy

  1. Log in to your Qualiphy clinic dashboard at app.qualiphy.me
  2. Navigate to Settings → API Settings
  3. Look for your API Key (a long alphanumeric string)
  4. Copy the entire key to your clipboard

Step 2: Enter Your API Key in Quidget

  1. In your WordPress admin, go to Qualiphy → Settings → General
  2. Find the Qualiphy API Key field
  3. Paste your API key into the field
  4. Click Verify & Save

Verification and Security

When you save your API key:

  • Verification: Quidget will test the connection to the Qualiphy API. This ensures your key is valid and your site can reach Qualiphy's servers.
  • Encryption: Your API key is encrypted using AES-256-CBC encryption before being stored in your database. It is never transmitted unencrypted and never exposed in logs.
  • Success Indicator: If verification succeeds, you'll see a green checkmark next to the API key field and a confirmation message.

If Verification Fails

If you see an error message:

  • Check the key: Ensure you've copied the entire API key correctly with no extra spaces
  • Check connectivity: Verify that your site can make outbound HTTPS requests to api.qualiphy.me
  • Check your Qualiphy account: Log into your Qualiphy dashboard to confirm your clinic account is active
  • Server configuration: If you use a firewall or security plugin, ensure it allows outbound requests to Qualiphy's API

Once your API key is verified, you're ready to run the setup wizard.

Running the Setup Wizard

After connecting your API key, the Quidget setup wizard will guide you through 11 configuration steps to prepare your store for consultations.

Step 1: Welcome & Terms

A welcome screen introducing the setup process and key configuration steps ahead.

Step 2: Requirements Check

Quidget verifies that your WordPress environment meets all system requirements (WordPress version, WooCommerce version, PHP version, SSL, API connectivity).

Step 3: Branding

Configure your clinic's visual identity:

  • Light Background Logo: Your clinic logo for use on light backgrounds (PNG recommended)
  • Dark Background Logo: Dark version of your logo for use on dark backgrounds
  • Primary Accent Color: Default #CFC9FF (light purple). Used for buttons, links, and highlights in the exam modal
  • Secondary Accent Color: Default #6e54a7 (dark purple). Used for text and secondary UI elements

These colors and logos appear throughout the patient experience: exam modal, thank-you pages, emails, and checkout.

Step 4: Environment Selection

Choose your Qualiphy environment:

  • Live: Production environment. Patients are billed for real consultations.
  • Sandbox: Test environment. Useful for testing the plugin before going live. Exams are processed but not billed.

Step 5: Custom Fields Setup

Configure additional patient information fields to collect at checkout (beyond the standard WooCommerce fields). These may include custom medical history questions, preferences, or clinic-specific requirements.

Step 6: Database Tables

Quidget creates database tables to store consultation data, pharmacy routing rules, and exam status. This step initializes those tables.

Step 7: API Key Connection

Confirmation that your API key has been successfully verified and connected.

Step 8: Service Area Configuration

Select the US states where you're licensed to provide consultations:

  • 48 states plus Washington, D.C. are available
  • Alabama and Mississippi are not currently supported
  • Your selection automatically creates a WooCommerce shipping zone for these states
  • Patients outside your service area will see "This product is not available in your area"

Step 9: Import Consultations

The wizard syncs your Qualiphy consultations and offers to create WooCommerce products for them. You can:

  • Create new products for each consultation
  • Link to existing WooCommerce products
  • Set pricing (with your configured margin applied)
  • Configure exam type (GFE or Rx) and pharmacy routing if applicable

Step 10: Configure Settings

Review and adjust key settings like auto-launch exams, exam launch mode (popup vs redirect), checkout behavior, and language/text customization.

Step 11: Complete

Setup wizard is complete. You're now ready to accept patient orders and deliver consultations.

Re-running the Wizard

You can re-run the setup wizard at any time by going to Qualiphy → Settings → General and clicking Run Setup Wizard. This won't overwrite existing settings—you can selectively update them as needed.

Your First Consultation

After completing the setup wizard, it's time to test the full consultation flow end-to-end. This ensures that exams are sent correctly, order lifecycle works as expected, and everything is ready for real patients.

Step 1: Access Your Storefront

  1. Navigate to your WordPress site's public storefront (not the admin)
  2. Find a consultation product you created during setup
  3. Click to view the product page

Step 2: Place a Test Order

  1. Click Add to Cart on the consultation product
  2. Proceed to checkout (you'll be redirected automatically if auto-redirect is enabled)
  3. Fill in the required patient information:
    • Gender (Male, Female, Non-binary, Prefer not to say)
    • Date of Birth
    • Phone Number
    • Email address (use one you have access to)
    • State (must be in your service area)
  4. Choose your payment method and complete payment
    • If in sandbox mode, use test credit card: 4242 4242 4242 4242, any future expiry, any CVC
    • If live, use a real payment method

Step 3: Verify the Exam Invite

Within seconds of completing payment, an exam invite email will be sent to the address provided at checkout:

  1. Check your inbox (including spam/promotions folders)
  2. Open the exam invite email and verify:
    • Your clinic logo and branding colors appear in the email
    • The email contains a link or button to start the exam
    • The patient's name and consultation type are correct
  3. Click the exam invite link to proceed

Step 4: Complete the Exam

  1. You'll be directed to the exam platform (hosted by Qualiphy)
  2. Fill out the patient questionnaire:
    • Demographic information
    • Medical history and health conditions
    • Current medications
    • Consultation-specific questions (dose preference, etc.)
  3. Submit the questionnaire
  4. You'll see a confirmation message with the provider's meeting link (for video consultation with a provider)

Step 5: Check Your Order in WordPress

  1. Go back to your WordPress admin
  2. Navigate to WooCommerce → Orders
  3. Find the test order you just created (look for your email or name)
  4. Click to open the order detail page
  5. Scroll to the Qualiphy Consultation section and verify:
    • Exam status is displayed (e.g., "Submitted", "Pending Review", "Approved")
    • Patient information matches what you entered at checkout
    • Consultation type is correct (GFE or Rx)
    • Provider name appears (if assigned)

Step 6: What Happens Next

In a real scenario, the provider would:

  • Review the patient's exam responses
  • Conduct a video consultation with the patient
  • Approve the exam (or defer if more information is needed)
  • For Rx consultations: Issue a prescription that gets routed to the patient's pharmacy
  • For GFE consultations: Complete the order (the patient receives clearance without a prescription)

In your test, you can simulate this by going to the Qualiphy app and approving the exam from the provider dashboard.

Troubleshooting

  • Exam invite not received: Check spam folders. If still not found, verify the email address you entered at checkout is correct, and check that your site can send emails properly.
  • Order not showing Qualiphy data: Refresh the page. If still blank, verify your API key is connected by checking Qualiphy → Settings → General.
  • Exam modal not opening or exam link not working: Clear your browser cache and cookies. Ensure JavaScript is enabled in your browser.
  • Payment declined: If using test mode, use the test card provided above. If live, check your payment gateway settings.

Success! If you've completed all steps, your Quidget installation is working correctly. You're ready to launch to patients. Explore the Plugin Settings guide to customize additional features like questionnaires, pharmacy routing, and patient communications.
